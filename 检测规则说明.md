# 投稿前 AI 数据质量控制系统：检测规则说明

本文档说明当前最新版软件会检查哪些问题、为什么这些问题值得复核，以及科研人员看到结果后应如何理解。

软件的判断结果不是“定性结论”，也不是对研究行为的直接判断。它只是把压缩包中的表格、图片和外部检查信息做自动筛查，找出需要人工回到原始记录、实验记录本、仪器导出文件、分析脚本和未裁剪原图中核对的位置。

## 风险颜色说明

软件只使用一套颜色风险系统：

| 颜色 | 含义 | 建议处理 |
|---|---|---|
| Red | 必须优先处理的问题 | 投稿前应逐项核对并解决，或形成清楚的书面解释。 |
| Orange | 需要回查原始记录的问题 | 建议回到原始数据、分析脚本、仪器导出文件中确认原因。 |
| Yellow | 建议记录说明的问题 | 通常风险较低，但建议保留解释，便于审稿或内部复核。 |

## 一、数值表格规则

这些规则主要检查 Excel、CSV 等表格中的数字模式。软件会尽量只针对数字内容判断，不把普通文字重复作为问题。

### N001：两列长期保持固定差值

检查内容：软件检查两列数字之间是否长期保持同一个差值。

为什么要查：固定差值可能来自合法的基线校正、背景扣除或单位处理；但如果没有处理说明，也可能是复制一列后统一加减某个常数得到的新列。

需要复核：确认这两列是否本来就应当存在固定差值关系；如果是合法处理，应在复核记录中说明处理方法。

高亮方式：通常高亮相关的两列，因为规则判断的是两列之间的整体关系。

### N002：两列可由简单直线关系解释

检查内容：软件检查一列是否几乎可以由另一列乘以固定系数、再加固定常数得到。

为什么要查：这种关系可能是合法换算、标准化或公式派生；但如果两列本应是独立测量结果，就需要警惕公式拖拽、复制修改或批量生成。

需要复核：查看分析脚本、公式或实验设计，确认两列是否应有这种数学关系。

高亮方式：通常高亮相关的两列。

### N003：多列小数尾部异常相同

检查内容：软件比较多列数字的小数部分，寻找小数尾部高度重复的情况。

为什么要查：不同测量值的小数尾部大量相同，可能来自复制小数部分、手工修改整数部分、格式设置异常或重复粘贴。

需要复核：回到原始导出文件，确认小数位是否由仪器精度、四舍五入或统一格式造成。

高亮方式：尽量高亮命中的单元格或相关列。

### N004：末位数字分布异常

检查内容：软件查看数字的末位有效数字是否过度集中在少数几个数字上。

为什么要查：真实测量值的末位数字通常不会长期集中在某几个数字；异常集中可能来自人工录入习惯、四舍五入、仪器精度设置或数据整理过程。

需要复核：确认仪器精度、记录方式、四舍五入规则是否能解释这种分布。

高亮方式：通常高亮触发该规则的变量列或相关单元格。

### N005：百分比和计数不匹配

检查内容：软件用分子、分母或总数重新计算百分比，检查是否与表中报告的百分比一致。

为什么要查：百分比与计数不一致，常见原因包括复制错单元格、分母引用错误、四舍五入规则不清或报告值未同步更新。

需要复核：优先核对报告中列出的不匹配行，确认百分比、分子、分母、总数来源。

高亮方式：高亮涉及计算的具体单元格。

### N006：重复的连续数值序列

检查内容：软件寻找一段连续数字序列是否在表格其他位置重复出现。

为什么要查：重复序列可能是同一批数据被合理复用，也可能是复制粘贴、模板区域未覆盖或重复导出。

需要复核：确认重复序列对应的样本、实验条件和数据来源是否相同。

高亮方式：高亮重复序列涉及的单元格或行。

### N007：数值列呈等差序列

检查内容：软件检查某列是否长期按固定步长增加或减少。

为什么要查：等差序列可能是时间点、剂量梯度、编号等设计变量；如果该列本应是实验测量值，则可能来自自动填充、排序编号或公式生成。

需要复核：确认该列是否是设计变量；如果不是，应回查录入和公式。

高亮方式：高亮命中的列或具体连续单元格。

### N008：某列大量重复同一个数值

检查内容：软件检查同一列中是否大量重复同一个数字。

为什么要查：大量重复值可能是检测下限、阴性/阳性对照、缺省值或合法分组；也可能是批量填充、缺失值替代或录入错误。

需要复核：确认重复值是否有实验设计或仪器输出依据。

高亮方式：高亮重复值所在的单元格。

### N009：均值、样本量和小数精度之间不匹配

检查内容：软件检查某些摘要统计值是否能由给定样本量和小数位精度合理产生。

为什么要查：在给定样本量和小数位时，某些均值只能由有限的原始整数计数产生；不匹配可能说明均值、样本量、小数位或录入内容存在问题。

需要复核：核对原始计数、均值计算方式和小数位设置。

高亮方式：高亮涉及均值、样本量或相关统计量的具体单元格。

### N010：不同文件中出现高度相似的数据块

检查内容：软件比较不同文件或不同 sheet 中的数值窗口，寻找高度相似的数据块。

为什么要查：不同文件复用同一数据有时合理，例如同一原始数据用于不同展示；但如果文件代表不同实验或不同条件，则可能是重复导出、误放文件或复制粘贴。

需要复核：确认这些文件是否确实应共享同一数据来源。

高亮方式：通常高亮相关数据块。

### N011：同一结果区的小数位数不一致

检查内容：软件检查同一结果区域内小数位数是否混用。

为什么要查：同一仪器、同一统计流程或同一导出格式通常会有相近的小数精度；混用精度可能提示手工修改、拼接不同来源数据或格式设置不一致。

需要复核：确认不同小数位是否有合理来源，例如不同仪器、不同单位或不同统计方法。

高亮方式：高亮小数位异常的单元格。

### 组合规则：同一区域同时触发多条数值规则

检查内容：如果同一文件、同一 sheet 或同一区域同时触发多条数值规则，软件会把它作为优先复核信号。

为什么要查：单一规则可能有合理解释；但多个不同规则集中在同一区域，说明该区域更值得优先回查。

需要复核：把该区域作为重点，逐项核对原始记录、分析脚本和导出文件。

## 二、基础表格质量规则

这些规则来自早期和通用表格检查逻辑，仍会作为辅助信号保留。

### 完全重复的数据行

检查内容：只比较数字内容，判断两行或多行是否完全相同。

为什么要查：完全重复的数字行可能来自重复录入、重复导出、样本编号错配或复制粘贴。

需要复核：核对这些行对应的样本编号、实验条件和原始记录。

高亮方式：高亮相关行，因为规则判断对象是整行。

### 高度相似的数据行

检查内容：只比较数字内容，判断两行是否在多数数字字段中高度相似。

为什么要查：高度相似可能来自相近实验条件，也可能来自复制一行后只修改少数数值。

需要复核：核对对应样本是否确实应高度相似。

高亮方式：高亮相关行。

### 完全相同的数据列

检查内容：判断两列数字是否逐行完全相同。

为什么要查：可能是同一指标重复展示，也可能是公式引用错列、复制整列或导出模板错误。

需要复核：确认两列变量名是否代表同一含义。

高亮方式：高亮相关列。

### 两列高度相关

检查内容：判断两列数字变化趋势是否几乎完全同步。

为什么要查：高度相关可能是正常生物学关系、单位换算或同一计算流程；若两列本应独立，则需要检查是否由复制、缩放或公式生成。

需要复核：结合实验设计判断两列是否本应高度相关。

高亮方式：高亮相关列。

### 固定倍数关系

检查内容：判断两列数字是否长期保持固定倍数。

为什么要查：固定倍数可能是合法单位换算或归一化；也可能是一列由另一列批量乘以常数得到。

需要复核：确认是否存在明确换算关系。

高亮方式：高亮相关列。

### 连续固定间隔变化

检查内容：判断某列连续数字是否按固定间隔变化。

为什么要查：可能是合法梯度或时间点；也可能是拖拽填充、排序编号或人工构造序列。

需要复核：确认该列是否为设计变量。

### p 值或 q 值超出合理范围

检查内容：检查 p 值或 q 值是否小于 0 或大于 1。

为什么要查：p/q 值理论范围是 0 到 1；超出范围通常说明导出列错误、格式转换错误或列含义误放。

需要复核：回到统计软件原始输出确认。

### p 值或 q 值为 0

检查内容：检查 p/q 值是否被记录为 0。

为什么要查：统计软件通常输出极小值而不是真正的 0；0 可能是显示位数过短、被截断或导出精度丢失。

需要复核：查看原始统计输出中的完整数值。

### fold change 小于或等于 0

检查内容：检查 fold change 是否为 0 或负数。

为什么要查：常规 fold change 通常应为正数；非正数可能说明列含义不是 fold change，或计算/导出过程有误。

需要复核：确认该列定义和计算方式。

### 丰度、计数或强度出现负数

检查内容：检查 abundance、count、intensity 等列是否出现负数。

为什么要查：这些值通常不应为负；负数可能来自背景扣除、标准化或错误替换。

需要复核：确认负数是否来自合法处理流程。

### 百分比超出 0 到 100

检查内容：检查百分比是否低于 0 或高于 100。

为什么要查：可能是比例和百分比单位混用，也可能是公式引用错误。

需要复核：确认该列使用的是 0-1 比例还是 0-100 百分比。

### 极大值或无穷大替代值

检查内容：检查是否存在异常大的数字或无穷大替代值。

为什么要查：常见于除以 0、计算失败、Inf 被替换成边界值或软件导出异常。

需要复核：回查计算公式和原始软件输出。

### 高缺失率的列或行

检查内容：检查某些列或行是否有大量空值。

为什么要查：可能是指标未检测、样本失败、导出不完整、合并表格错位或样本信息未填写完整。

需要复核：确认缺失是否符合实验设计或数据采集情况。

### 富集分析 Count 与 Genes 数量不一致

检查内容：在富集分析表中，检查 Count 是否等于 Genes 列实际列出的基因数量。

为什么要查：不一致可能来自复制错误、分隔符错误、筛选条件变化后未更新 Count。

需要复核：回查富集分析原始结果和导出文件。

### 富集分析条目为空或重复

检查内容：检查富集分析 term 名称是否为空，或同一条目是否重复出现。

为什么要查：可能是导出错误、合并表格错误或重复筛选。

需要复核：确认富集分析软件原始输出。

## 三、附表区块审计规则

这一部分整合了工作目录下 `data_audit.py` 的思路，主要检查补充表、figure source data 或类似表格中的“数值区块”。数值区块可以理解为表中一片连续的数字区域。

### 两个数值区块完全相同

检查内容：比较两个数字区块是否完全一致。

为什么要查：完全相同的数据块可能是合理重复展示，也可能是复制粘贴或重复导出。

需要复核：确认两个区块是否本应相同；如果代表不同实验条件，应重点回查。

### 两个数值区块存在固定倍数关系

检查内容：检查两个数字区块是否整体保持固定倍数。

为什么要查：可能是单位换算或归一化；也可能是批量乘以常数生成。

需要复核：确认是否有明确换算公式或处理流程。

### 两个数值区块存在固定差值关系

检查内容：检查两个数字区块是否整体保持固定差值。

为什么要查：可能是基线校正、平移处理或批量加减常数。

需要复核：确认处理流程是否合理。

### 同一区块内有完全重复的行或列

检查内容：检查同一个数值区块内部是否存在完全重复的数字行或数字列。

为什么要查：可能是重复样本、重复指标、模板未覆盖或复制粘贴。

需要复核：核对对应样本或指标是否本应重复。

### 大表或区块过多时跳过部分审计

检查内容：如果表格太大或数值区块太多，软件会跳过部分两两比较。

为什么要查：这是为了避免软件运行时间过长，不代表这些区域没有问题。

需要复核：如果该大表是关键数据，建议按实验分组拆分后重新检查。

## 四、图片规则

图片规则会扫描压缩包中的图片文件，检查完全重复和视觉高度相似的图片。它不能替代专业图像取证平台，也不能替代人工查看未裁剪原图。

### I001：完全相同的图片文件

检查内容：判断两个图片文件内容是否完全一致。

为什么要查：完全相同图片如果代表不同实验条件，可能需要回查图片来源；如果只是同一图片在不同位置重复引用，则应在材料中说明。

需要复核：打开原始未裁剪图片，核对文件来源、实验条件和图注。

### I002：视觉上高度相似的图片

检查内容：使用感知哈希等方式寻找视觉上高度相似的图片。

为什么要查：相似图片可能来自同一视野、轻微裁剪、压缩或合理复用；也可能是不同实验条件误用了相近图片。

需要复核：人工比对原始图片、实验条件和图版来源。

## 五、表格解析和外部工具状态规则

### 表格文件解析失败

检查内容：软件无法读取某个 Excel、CSV 或类似表格。

为什么要查：文件可能损坏、加密、格式不受支持，或内容不是标准表格。

需要复核：手动打开文件，确认文件是否正常；必要时另存为标准 xlsx 或 csv 后重新检查。

### 外部 AI 工具需要手动检查

检查内容：如果没有配置外部 API，或外部工具需要用户手动上传检查包，软件会记录该状态。

为什么要查：这不是数据错误，只是提醒外部图片或数据平台没有自动完成检查。

需要复核：如需要外部平台结果，可手动上传软件生成的图片检查包，或导入外部工具报告。

### 外部检查报告提示风险

检查内容：如果用户导入了外部图片或数据检查报告，软件会把其中的风险提示纳入总报告。

为什么要查：外部工具可能发现本地规则无法覆盖的问题。

需要复核：以外部平台原始报告为准，结合软件报告中的位置提示人工确认。

## 六、结果使用建议

1. 先看 Red，再看 Orange，最后看 Yellow。
2. 不要只看问题数量，要看问题是否集中在关键图表、关键补充表或关键结论数据中。
3. 对于列关系规则，例如高度相关、固定倍数、固定差值，软件高亮整列是合理的，因为规则判断的是两列整体关系。
4. 对于计算错误、百分比不匹配、重复值等可以定位到单元格的问题，软件会尽量只高亮具体单元格。
5. 对于行级问题，例如重复行或高度相似行，软件高亮整行是合理的，因为判断对象是整行。
6. 如果某条规则没有对应 sheet 显示，常见原因是该问题来自图片、外部报告、解析失败文件，或该问题无法精确映射回单元格。
7. 所有问题都应结合实验设计判断。合理的单位换算、归一化、仪器精度、设计梯度、阴性对照、检测下限，都可能解释某些规则信号。
